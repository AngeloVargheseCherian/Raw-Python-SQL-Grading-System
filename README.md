# Raw-Python-SQL-Grading-System

Student Grading System (A Journey in Raw Logic)

This is my Class 12 Computer Science project, built between Class 11 and Class 12. It represents my first major dive into software development, created entirely from scratch using my own intuition and the fundamentals I learned in school.

🧠 The Story Behind the Code
I started this project in Class 11 when I first began learning Python. At that time, it was just about figuring out how to make logic work. By Class 12, as we were introduced to SQL, I added the database layer to make the data persistent.

The most important thing about this repository is that it shows my raw thinking. I didn't use external tutorials or frameworks; I simply built it "on top and top" as I learned new concepts.

Intuition-Driven: Every if statement and function was coded based on how I thought a grading system should work.

Pure Scratch-Build: No outside help or copied templates.

The Joy of Coding: This project was a fun personal challenge for me to see if I could bridge the gap between Python logic and SQL storage.

🛠️ How it Works
The application is a console-based management system that handles the "Full Cycle" of student data:

The Logic Layer: I built a custom grading scale that maps numerical marks to specific grades (A1, B1, etc.).

The SQL Layer: I integrated MySQL to ensure that the data stays saved even after the program closes.

The User Flow: I designed a simple CLI (Command Line Interface) with a login gate and a menu to Insert, Show, Delete, or Modify records.

📜 Key Features
Dynamic Modifications: Instead of just rewriting a whole row, I built specific functions (modname, modmark, etc.) to target and change individual pieces of data.

Relational Interaction: The code handles the communication between Python variables and SQL tables using mysql-connector.

Custom Auth: A basic password-protection loop I designed to simulate how a real application might restrict access.
